# PHP

# 1. 定位Bug - 投票功能 strtotime('+1 month') 时间计算错误

发布时间：Mar 17, 2019 by FamousMai

# 發現bug

中午一觉醒来，群里说投票功能的结束时间出错了，整整多出了30天左右

# 定位相關代碼

爲啥之前沒出現這種問題，到了今天才出現了，看了看代碼

/*
 * 获取本期投票功能结束时间，即下一期开始时间
 */
function getPollEndTime(){
    // 本月5号
    $month_fifth = strtotime(date("Y-m-05 12:00:00"));
    // 下个月5号
    $next_month_fifth = strtotime(date("Y-m-05 12:00:00",strtotime('+1 month')));
    $now_time = time();


    // 若本月5号已经过去，则结束时间为下月5号
    if ($now_time >= $month_fifth) {
        return $next_month_fifth;
    }
    // 若本月5号还未过去，则结束时间为本月5号
    if ($now_time < $month_fifth) {
        return $month_fifth;
    }
}

# 找出問題所在

乍一看，沒啥毛病啊，於是斷點調試一下了這個獲取結束時間函數，發現了原來算到到下下個月的5號去了，實際上9月5號為結束時間，變成了10月5號為結束時間，所以才多處了那30多天。 思考問題原因

問題找到了，可爲什麽到了今天才計算出錯呢，看了看今天的日期8月31號，月末，猜想會不會strtotime('+1 month')這種計算方式有問題呢 確認問題原因

# 百度了一下，果然是這樣

strotime計算上月下月問題 (opens new window)

# 修復BUG

date("Y-m-05 12:00:00",strtotime('+1 month'))

修改為

date ("Y-m-05 H:i:s", mktime(12, 0, 0, date("m") + 1, 1, date("Y")))

# 測試

時間正常，bug修復

# 其他

原先的改法確實比較複雜，可以這樣來改，但是不知道怎麽解釋才好

// 下個月五號 $next_month_fifth = strtotime(date("Y-m-05 12:00:00", strtotime('+1 month')));

=>

// 本月五號 $month_fifth = strtotime(date("Y-m-05 12:00:00")); // 下個月五號 $next_month_fifth = strtotime(date('Y-m-05 12:00:00' , strtotime('+1 month' , $month_fifth)));

# 总结

要使用strotime來計算跨月的時間，還是要慎用，strotime函數跨月本身計算會有問題
